## 2. レビューの重点観点

特に以下の点について、不備や改善余地がないかを厳しくチェックしてください。

### ① アーキテクチャとモジュール設計
- `src/extension.ts` はコマンドを一部切り出したものの、依然として多くのモジュールをインポートし、肥大化しています。
- パネルのライフサイクル管理、各種レシーバー/ハンドラー、エージェント初期化などをさらに別モジュール（例: `panelManager.ts` など）へ疎結合に分離するための具体的なリファクタリング計画を提案してください。

### ② WebSocket & ネットワークセキュリティ (`src/remotePlayServer.ts`)
- 自作解析から `ws` パッケージへの移行により安全性は向上しましたが、接続管理に穴はありませんか？
- **接続数制限 (maxClients)** や **接続維持/タイムアウト監視 (Ping/Pong)**、認証前接続の強制切断タイマー (`authTimer`) が正しく機能し、DOS攻撃やメモリリークを引き起こさないようになっているか。
- トークンの検証処理やログマスク処理に不備はないか。
- クライアントから送られてくるメッセージ（JSONパース時など）で、サーバーがクラッシュする脆弱性はないか。

### ③ I/Oとファイル操作の非同期化
- `fs.readFileSync` から `fs.promises.readFile` への変更は適切に行われているか。
- `game_state.json` のアトミックな読み書きや、競合状態（Race Condition）を防ぐ仕組みになっているか。

### ④ 今後の拡張への備え
- 今後予定されている **Phase 4A: VLM (Vision-Language Model) 統合（画面の画像認識など）** や **バックグラウンドシミュレーションの強化** に向けて、現在の `extension.ts` や状態同期の仕組みでボトルネックになりそうな箇所があれば指摘してください。
